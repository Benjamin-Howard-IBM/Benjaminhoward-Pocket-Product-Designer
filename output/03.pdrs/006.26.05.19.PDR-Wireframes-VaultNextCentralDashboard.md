# Vault Next Central Dashboard - Wireframes

Depends on: ../02.strategy/Vault-Next-Combined.md, 005.26.05.19.PDR-WireframePlan-VaultNextDashboards.md

## Persona served

Primary: Platform Engineer. Lands here after sign-in at the Organization scope to get a fast read on what is configured and what needs attention.

Secondary: System Security Engineer. Uses the same surface to confirm posture before drilling into Cryptography.

Application Developer is not a primary user of this dashboard. Their JTBD is served by per-resource browsing.

## Day 0 goals

- Single-screen summary of Secrets + Cryptography at the selected scope.
- Counts only. No trend, no time-series.
- Deep-link entry points to the two specialist dashboards.
- Light reference to Access Control via Auth methods and Identities counts in the footer panel.

| Cryptography panel - By state | Cryptography Metrics > # of Keys by state (active / decrypt-only / disabled / pending deletion) |
| Clients tile row (Human / Machine / Agent) | Secrets Metrics > # Clients by type, reconciled with Users/Entities/Identities > # Clients by type. Taxonomy reconciliation is Open Question 1 in 005. |

Out of scope and not shown: CREATED BY, LAST ACCESSED (Secrets CSV out-of-scope note), all operational traffic metrics (CSV "ALL OPERATIONAL METRICS OUT OF HASHI SCOPE").

## Day 0 interactions

- Scope change: changing Organization or Project dropdown refetches all tiles and panels. No partial states.
- Tile click: Total Secrets and # Secret Stores route to the Secrets dashboard with Project filter preserved. Total Keys and # Key Stores route to Cryptography with Project filter preserved.
- "View dashboard ->" ghost button: same routing as the tile click. Provided as an explicit affordance for keyboard users and to make the deep-link discoverable.
- Empty state at fresh org (no projects, no stores): each tile shows "0" with a Carbon `EmptyState` line under the panel body offering "Create a project" or "Create a store" depending on the panel. No `InlineNotification` at Day 0.
- No time-range. No date selection.

## Day 1 goals

- Add temporal context (sparklines, period comparison) without changing the page's information architecture.
- Surface what needs attention up front.
- Provide a top-stores-by-activity panel as a gated extension once operational metrics ship.

## Day 1 interactions

- Time-range picker (`DatePicker` range): changes the period for sparklines, trend lines, and the comparison strip. Tile primary values still reflect "now" (current count); the delta and sparkline reflect the selected range.
- Prometheus-gated controls: if the time-series store is not configured, the picker is disabled, the `InlineNotification` appears under the filter bar, sparklines are hidden, trend lines are replaced with an `EmptyState`, and the comparison strip body collapses to a single-line empty state.
- Attention panel: each row routes to the relevant filtered list. Keys pending deletion -> Cryptography dashboard with state filter applied. Secrets expiring -> Secrets dashboard with TTL filter applied.
- "Review all ->" ghost button: routes to a combined attention queue (out of scope for Day 1 wireframes; treat as future surface).
- Top stores panel empty state: title "No activity data yet"; body "Configure a time-series datastore to enable operational metrics."; single ghost Button "Go to Manage scopes".

## Carbon mapping

Components used:

- `UI Shell` (Header, SideNav, HeaderGlobalAction).
- `Breadcrumb` + page title.
- `Dropdown` (Organization, Project).
- `DatePicker` range (Day 1 only).
- `Tag` for active filters.
- `Tile` (clickable variant for the 4-up row; non-interactive for summary panels and footer tiles).
- `Button` ghost for "View dashboard ->" and "Review all ->".
- `SimpleBarChart`, `DonutChart` for Day 0 breakdowns.
- `LineChart` for Day 1 trends (small inline variant for sparklines).
- `InlineNotification` (kind=info) for gating copy.
- `EmptyState` (Carbon for Products) for zero-data panels.

Token slots:

- Surface background `$background`.
- Tile and panel surface `$layer-01`; hover `$layer-hover-01`; subtle border `$border-subtle-01`.
- Numeric value `$text-primary`; tile label and caption `$text-secondary`; helper / empty-state copy `$text-helper`.
- Attention panel uses `$layer-02` to visually separate from tile row without color.
- Spacing: tile inner padding `$spacing-05`; vertical gutter between sections `$spacing-07`; horizontal gutter inherits the Carbon 16-col grid.

Grid columns (Carbon 16-column):

- Tile row (4 tiles): 4 / 4 / 4 / 4.
- Attention panel: 16.
- Side-by-side summary panels: 8 / 8.
- Clients tile row (3 tiles): 4 / 4 / 4, with 4 columns reserved on the right (Day 0) or used for the Auth methods tile (Day 1 layout variant).
- Top stores panel: 16.
- Comparison strip: 16.
- Footer tiles (Auth methods, Identities): 8 / 8.

## Open questions / assumptions

- Assumption: "Identities" tile at Day 0 uses Auth methods CSV row "user mapping" as the proxy source. Confirm whether identities count is sourced from the IAM service directly.
- Assumption: Key Stores share the same status taxonomy as Secret Stores (available / pending deletion). Confirm with engineering.
- Open: Should the Attention panel sit above or below the tile row? Day 1 ASCII places it above to match the persona's job-to-be-done. Validate with Platform Engineer in usability review.
- Open: Sparkline range when the user has not opened the date picker. Default to 7d. Confirm.
- Open: Per-tile delta sign and direction. "+24" implies more secrets is good. For "pending deletion" deltas, a positive delta is bad. Need a status convention (e.g., gray neutral now; in higher-fidelity revisions, add semantic color).
